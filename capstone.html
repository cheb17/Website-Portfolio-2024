Short description about the application

The study aimed to develop a mobile application that monitors basketball players' shooting percentages and identifies missed shots easily and accurately and would want to include a proper visualization of the complete output every quarter of the game with an easy plot of touch coordinates that will be saved into the database. The study aims to improve the future development of coaches, trainers, and players in terms of game analysis and game progress, analyzing each player's strengths and weaknesses regarding shot placement and giving more information on both team and individual players. Not only that, it can be used by players for their player growth, to show what they can improve on. Additionally, developers aim to improve a mobile application with all the functionality other apps lack. It will mainly focus on the made or missed shots on the court and visualization of the overall game status.





Main Screen



Sign In - where the user can log in to their account



Sign Up - where the user can create their account



Home Screen - where the user can allow to create, review, change password and sign out of the app.



Create the Match Details - where the user inputs the Team and Opponents name.



Plotting of Data - where the user can start plotting the missed and made shots for each quarter, that will then be used for the clustering analysis.



Records - where the user can review back all the matches that have been created.



Sample of the 4th Quarter Visualization and Game Statistics - where the user can see the missed and made shots in the court, including the game Statistics of the current quarter.



Clustering - where the user can see the clustering result of the missed and made shots. So, that they can analyze which areas on the court the players need to improve.
